# Ivideon Sea Battle — онлайн-версия с несколькими играми

## Что добавлено

- 13 администраторов в готовой SQL-миграции.
- Каждый администратор может создать отдельную игру.
- У каждой игры свой набор показателей для выстрела.
- Показатели можно добавлять, переименовывать, включать, выключать, делать обязательными и задавать для них порог.
- Публичное табло показывает все опубликованные игры.
- Каждый посетитель выбирает игру в выпадающем списке; выбор сохраняется на его устройстве.
- Игры хранятся в отдельных строках Supabase, чтобы администраторы могли параллельно вести разные игры с меньшим риском перезаписи данных.
- Старая строка `main` автоматически переносится в новый формат при первом входе в обновлённую админку.

## Файлы

- `index.html` — публичное онлайн-табло.
- `admin.html` — админка с авторизацией Supabase.
- `config.js` — Project URL и publishable key.
- `multi-admin-migration.sql` — список администраторов и проверка доступа.
- `.nojekyll` — настройка GitHub Pages.

## Обновление GitHub

1. Распакуйте архив.
2. Откройте репозиторий `https://github.com/uziuzborn/ivideon-seabattle`.
3. Нажмите **Add file → Upload files**.
4. Перетащите содержимое папки в корень репозитория.
5. Подтвердите замену существующих файлов.
6. Выберите **Commit directly to the main branch**.
7. Нажмите **Commit changes**.

GitHub Pages повторно публикует сайт из ветки `main` автоматически.

## Настройка Supabase

### 1. Запустить миграцию

1. Откройте Supabase → **SQL Editor → New query**.
2. Откройте файл `multi-admin-migration.sql`.
3. Скопируйте код целиком.
4. Нажмите **Run**.
5. В результате появится таблица со статусами email.

### 2. Создать Auth-пользователей

SQL добавляет адреса в список разрешённых администраторов, но не создаёт им пароли.

Для каждого адреса:

1. Откройте **Authentication → Users**.
2. Нажмите **Add user → Create new user**.
3. Введите email.
4. Укажите пароль.
5. Включите **Auto Confirm User**, если переключатель доступен.
6. Сохраните.

## Как администратору создать свою игру

1. Войти в `admin.html` под своей почтой.
2. Открыть вкладку **⑤ Игры**.
3. Ввести название и нажать **Инициировать новую игру**.
4. Перейти в **① Настройка**.
5. В блоке **Параметры для получения выстрела**:
   - переименовать стандартные показатели;
   - выключить ненужные;
   - добавить новые;
   - отметить обязательные;
   - включить порог и указать минимальное значение.
6. Добавить участников, расставить флот и запечатать поле.
7. Во вкладке **② Выстрел недели** появятся только включённые показатели этой игры.

## Как посетителю выбрать игру

На публичном табло в шапке есть список **Игра**. Посетитель выбирает нужную игру, и выбор сохраняется в его браузере. Изменения рейтинга приходят через Supabase Realtime.

## Безопасность

В репозитории находится только publishable key. Доступ на запись ограничен RLS и функцией `is_seabattle_admin()`.

Не загружайте в GitHub:

- `service_role`;
- secret key;
- пароль базы данных;
- пароли пользователей.

Одинаковый пароль для всех технически возможен, но безопаснее выдать временный пароль и затем заменить его на индивидуальный.

## Безопасный вход в админку

После авторизации админка всегда открывает вкладку **«③ Статистика»**. Это снижает риск случайно показать расстановку кораблей во время демонстрации экрана. Вкладка **«① Настройка»** открывается только после осознанного нажатия администратора.
